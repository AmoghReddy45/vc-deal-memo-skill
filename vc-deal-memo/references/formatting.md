# Formatting and Presentation Reference

Use this when producing DOCX, PDF, or Google Docs-targeted memos.

## General Formatting

Default:

- Font: Arial.
- Body: 10-10.5 pt.
- Section headers: black or very dark navy bars with white bold text, or a consistent light gray table-header style if the memo leans minimal.
- Tables: thin borders, usually 0.5 pt. Avoid heavy gridlines.
- Header rows: consistent across all tables.
- Body copy: compact paragraphs, not dense walls of text.
- Sources: separate page or compact appendix; do not crowd the final recommendation.

## Tables

Use content-sized columns when columns have unequal density.

Good uses:

- deal terms,
- investment history,
- comps,
- financial snapshot,
- risk/watch-items,
- product mix,
- round details.

Avoid:

- equal-width columns when one column is a short label and another is long reasoning,
- right-aligned text headers unless all numeric,
- oversized rank columns,
- thick borders,
- inconsistent header fills.

For a top "Deal" table, a clean convention:

- "The Deal:" as text above the table.
- Left column light gray.
- Right column slightly lighter gray.
- No inner borders if visually clean.
- Thin perimeter border.

## Charts

Charts should earn space in 3 seconds.

Useful chart types:

- reported run-rate growth,
- revenue by year,
- product-line mix,
- use of funds,
- comp valuation multiple.

Always add a basis footnote:

> Source: company-provided materials. Gross annual run-rate is annualized current-period activity, not GAAP revenue or contracted SaaS ARR.

## Layout QA

Before finalizing:

- Render DOCX to page images or PDF.
- Check that section headers are not orphaned.
- Check charts/images render in the target format.
- Check page breaks around tables.
- Check title size and weight.
- Check header/footer/logo imports correctly if using Google Docs.
- If images break in Google Docs, use embedded body images and avoid fragile headers or linked images.
